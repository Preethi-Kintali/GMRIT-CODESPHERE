# GMRIT CodeSphere — v3 Product Roadmap & Evolution Plan

This document outlines the strategic vision and technical objectives for the `v3` update of the GMRIT CodeSphere platform. Building upon the solid foundation of v2 (Admin Systems, Notifications, and Profiles), v3 focuses on **AI Intelligence**, **Advanced Analytics**, and **Collaborative Interviewing Tools**.

---

## 1. AI-Powered Intelligence (CodeSphere AI)
The core theme of v3 is making the platform "smarter" for both interviewers and candidates.
- [ ] **Automated Post-Interview Summaries:**
  - Leverage LLMs (OpenAI/Gemini) to analyze the final code submission.
  - Generate a "Candidate Competency Report" covering: Problem Solving, Code Quality, Time Complexity, and Optimization suggestions.
- [ ] **Dynamic AI Hints:**
  - Interviewers can toggle "AI Hints" for candidates.
  - Candidates can request up to 3 hints per session (configurable by Admin), which the AI provides without giving away the full solution.
- [ ] **Real-time Complexity Analysis:**
  - Integrated sidebar in the editor showing Big O (Time/Space) estimates as the candidate types.

## 2. Session Playback & Forensic Review
Enhancing the review process by moving beyond static final code snapshots.
- [ ] **Keystroke Replay Engine:**
  - Implement delta-syncing for code changes in MongoDB/S3.
  - Develop a "Playback" UI for Admins/Interviewers to watch the entire coding process at 1x/2x/4x speeds.
- [ ] **Cognitive Flow Visualization:**
  - Chart "Thinking Time" vs. "Coding Time" to help interviewers see where the candidate struggled most.
- [ ] **Time-stamped Internal Notes:**
  - Interviewers can leave private comments at specific timestamps during the live session for later review.

## 3. Collaborative Tools & Extended Functionality
Expanding the platform beyond just a "Code Editor".
- [ ] **System Design Whiteboard:**
  - Integrate a collaborative drawing canvas (Excalidraw-like) for system design rounds.
  - Persistent state saved alongside the session code.
- [ ] **Rich Test Suite Support:**
  - Transition from single-input execution to a "Test Case Runner".
  - Admins can define public and hidden test cases for each problem.
- [ ] **Enterprise Proctoring 2.0:**
  - **Presence Detection:** Use Stream.io video analytics to detect if a candidate leaves the frame.
  - **Browser Lockdown Mode:** Optional "Full Workspace" mode that prevents exit without permission.

## 4. Platform Scalability & UX Polish
- [ ] **Universal Code Themes:**
  - Support for custom editor themes (Tokyo Night, Dracula, Github Dark) stored in user preferences.
  - Dynamic Glassmorphic UI updates with Framer Motion animations.
- [ ] **Advanced Admin Dashboard:**
  - Aggregated metrics: "Average Success Rate per Problem", "Interviewer Activity Heatmaps", and "Candidate Onboarding Funnels".
- [ ] **Mobile-Friendly "Admin Mode":**
  - Allow Admins to manage interviews and view schedules via a dedicated mobile-responsive layout.

---

## 5. Technical Stack Upgrades
- **Frontend:** Transition to `Framer Motion` for all transitions; Explore `WebWorkers` for intensive code analysis.
- **Backend:** Migrate background tasks to `Inngest` fully (v2 used it partially); Implement `Redis` for caching session snapshots for playback.
- **Infrastructure:** S3 integration for storing large playback logs and resume uploads.
